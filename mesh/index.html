tmux for coding-agent swarms

Keep every agent visible.

mesh is a small tmux wrapper for local coding agents. The human stays in control, the lead stays visible, and every worker remains a real terminal you can watch, interrupt, and steer.

Install mesh View source
lead
$ curl -fsSL https://het-patel.dev/mesh/install.sh | bash
$ tmux
$ codex --ask-for-approval never
$ mesh-codex --count 5 --replace
$ mesh assign codex-01 "Own auth tests and report back."
$ mesh show agents
worker awareness
$ mesh context codex-01 --lines 20
$ mesh changes codex-01 --lines 20
$ mesh read codex-01 --lines 80

Human-first

The human can click into any pane, type directly, and stay in charge of the swarm.

Small surface area

No daemon, no database, no hidden orchestration layer. Just tmux, a config layer, and a bridge.

Agent-aware

Lead agents can read sibling context, inspect repo changes, and hand off work with live swarm state attached.

Install

Install mesh directly from your site, or clone the repo if you want local development.

public install
curl -fsSL https://het-patel.dev/mesh/install.sh | bash

Clone install

git clone https://github.com/hetpatel-11/mesh.git
cd mesh
bash ./install.sh

What it installs

  • ~/.local/bin/mesh
  • ~/.local/bin/mesh-codex
  • ~/.local/bin/mesh-codex-open
  • ~/.mesh/tmux.conf
  • ~/.mesh/skills/mesh/SKILL.md

Common flows

These are the commands people will actually use day to day.

Start a workspace

mesh workspace --session agents --count 10 --cmd zsh

Show the live session

mesh show agents

All-Codex swarm

mesh-codex --count 5 --replace

Codex sandbox workaround

mesh-codex-open --count 5 --replace

Skills

Humans do not need a skill to use mesh. The optional skill is for coding agents like Claude Code or Codex so they follow the mesh workflow instead of inventing bad fallbacks.

  1. Install mesh with the standard install command.
  2. Point your agent to the installed skill file at ~/.mesh/skills/mesh/SKILL.md.
  3. If your agent has its own skill loader, install that file or folder there.
  4. For raw file access, the public skill path is /mesh/skills/mesh/SKILL.md.
